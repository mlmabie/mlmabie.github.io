RAM Labs Program Dossier

Operational AI/ML work across tactical cybersecurity, distributed inference, and automated software repair

Condensed from internal-facing program descriptions and technical biographies

At RAM Labs, my work sat at the intersection of machine learning research and deployment realism: nonstationary data, constrained hardware, tactical operating conditions, and analyst trust requirements. The common objective was to produce systems that remained useful when assumptions broke.

T-UEBA (Tactical User and Entity Behavior Analytics)

Mission: support Zero Trust enforcement in tactical networks where context shifts rapidly and static baselines fail.

Technical profile: heterogeneous temporal graph modeling, multi-plane risk/energy formulation, adaptive thresholds, and analyst feedback loops.

Deployment posture: Elastic-integrated, dual-mode architecture for connected/disconnected operation in resource-constrained environments.

DL-PATCHER

Mission: automate software vulnerability repair from code corpora at scale.

Technical profile: seq2seq and transformer-family models (including GraphCodeBERT), code-oriented preprocessing pipelines, and pretraining/fine-tuning workflows for patch generation.

Output: practical repair-oriented modeling plus patent-linked research trajectory.

SpHyRE-Net (DARPA HyDDENN)

Mission: low-latency AI capability on SWaP-constrained edge devices.

Technical profile: sparse/distributed representations oriented toward low-shot learning and anomaly detection under limited labels.

DAICON

Mission: distributed AI at the tactical edge across heterogeneous nodes without dependence on homogeneous radio stacks.

Technical profile: coded matrix computation, fused partitioning, NDN-driven peer-to-peer distribution, and resilient execution under ad hoc network conditions.

DEVIS

Mission: detect embedded vulnerabilities in compiled binaries.

Technical profile: ML-assisted path prioritization layered over instrumentation/fuzzing workflows, with synthetic data generation from Juliet/LAVA/GitHub-style sources.

Role Scope and Patent Context

My role spanned model design, data engineering, low-resource training strategy, and deployment-minded integration. Program work included AIxCC-adjacent vulnerability detection/repair exploration, distributed edge inference investigations, and technical proposal architecture.

Patent linkage: Automated Bug Fixing Using Deep Learning Including Pre-training and Fine-tuning, U.S. application 18/375,839 (Geddes, Mabie, McGraw).
