Research Thesis Map

Representation boundaries, calibrated action, and learned residuals

The learned part should not carry the whole burden of the system. My recurring research question is structural: what should remain explicit, what should become an object, what should become a relation, what should be validated deterministically, and what ambiguity is actually worth learning.

1) Cognitive security and calibrated action

In contested environments, anomaly detection without confidence and context is operationally brittle. The target is calibrated inference with analyst-legible evidence paths, not alert volume or abstract embedding quality.

Cognitive Security brief · RAM Labs dossier

2) Graph modeling and representation boundaries

Graph methods are strongest when they do not pretend to be the whole stack. In cyber this means temporal heterogeneity and evidence drilldown. In spatial systems this means geometry-native object identity, typed relations, edit semantics, and deterministic validators. The lesson comes as much from fixtures and CAD as from papers.

Graph Modeling · Augrade framing

3) Thin control planes for stronger models

As model capability scales, orchestration should simplify. Good harnesses preserve observability, provenance, evaluation, and operator control while minimizing bespoke glue code. The system should expose uncertainty, not perform confidence.

Current position · Compressed forms

4) Current physical-AI stack

For CAD and AEC workflows, the stack I keep returning to is: exact artifact → stable object state → typed relation graph → multiscale compression → hard validators → learned residual repair → review loop.
